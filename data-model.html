Agent Configuration — Full Data Model

Each agent record = 5 entities. Skills follow the agentskills.io specification.

AgentRecord Root Entity
id
number
required Unique record identifier
agent
AgentConfig
required → see AgentConfig
managedSkills
Skill[]
array agent skills spec Ordered SKILL.md entries
knowledge
KnowledgeDoc[]
array Unstructured docs for RAG retrieval
AgentConfig Agent Identity & Runtime
Identity
name
string
required Human-readable agent name
description
string
required What this agent does
url
string
required Agent endpoint URL
version
string
required Semantic version (e.g. 2.1.0)
Capabilities
capabilities.streaming
boolean
Supports streaming responses
capabilities.async_execution
boolean
Supports async task execution
Registered Tools (runtime endpoints)
tools[].id
string
Tool endpoint identifier
tools[].name
string
Display name
tools[].description
string
What the tool endpoint does
tools[].tags
string[]
array Categorization tags
Handler
handler
Handler
required → see Handler
Handler Request Processing
class
string
required Fully qualified handler class
args.agent_executor
string
required Executor class path
args.task_store
string
required Task persistence (InMemory / Redis)
Skill
agentskills.io SKILL.md
UI-only fields (not in spec)
id
string
ui Internal record ID (sk-xxxxx)
status
enum
ui "active" | "inactive"
lastModified
ISO 8601
ui Last edit timestamp
Spec required — YAML frontmatter
name
string(1-64)
required spec Lowercase slug (a-z, 0-9, hyphens). No leading/trailing/consecutive hyphens. = directory name.
description
string(1-1024)
required spec What it does + when to use it. Include keywords for agent discovery.
Spec optional — YAML frontmatter
license
string | null
optional spec License name or file ref (e.g. Apache-2.0)
compatibility
string(0-500)|null
optional spec Environment reqs (product, packages, network)
metadata
{key,value}[]
optional spec array Arbitrary key-value pairs (author, version, ...)
allowedTools
string | null
optional spec Space-delimited pre-approved tools (experimental)
Full SKILL.md file
content
string
required Complete SKILL.md = YAML frontmatter + markdown body (instructions, examples, edge cases)
KnowledgeDoc RAG Document
id
string
required Unique doc ID (kb-xxxxx)
name
string
required Filename or display title
type
enum
required "pdf" | "docx" | "txt" | "md" | "url" | "text"
source
string
required "file" for uploads, or the full URL string
uploadDate
ISO 8601
required When the document was added
status
enum
required "indexed" | "processing" | "failed"
size
number | null
optional File size in bytes (null for URLs)
tags
string[]
optional array Labels for traceability / filtering
